Unit 1: Introduction to Wireless Communication Systems

This unit introduces the fundamental concepts of wireless and mobile communication, tracing its evolution and outlining the basic architecture of cellular systems. Understanding these basics is crucial for grasping more advanced topics in wireless technology.

A. Defining Mobile and Wireless Communication

Wireless communication is the transmission of information over distances without physical conductors (wires/cables). Mobile communication enables communication between individuals in different places without physical connections, using devices like smartphones that connect wirelessly, typically via radio waves.

The first commercial mobile telephony began in 1946 (BELL, USA), using a single high-power analog transmitter. "Mobile" often refers to terminals in high-speed platforms (e.g., car phones), while "portable" refers to handheld terminals for walking speeds.

B. Limitations of Conventional Mobile Telephone Systems

Early systems suffered from:

  • Limited Service Capability: No handoff between zones; calls dropped when moving. Fixed channels per zone limited user capacity.
  • Poor Performance: Very few channels for large areas (e.g., 33 channels for a 50-mile diameter area in early MTS/IMTS). High call blocking in cities like New York by 1976.
  • Inefficient Spectrum Utilization: One channel per customer across the entire coverage area, leading to underutilized spectrum.

These limitations, along with advances in LSI/microprocessor tech, drove the need for the cellular concept.

C. Basic Cellular System Architecture

A cellular system consists of the Mobile Station (MS), Base Station (BS), and Mobile Switching Center (MSC/MTSO).

Mobile Station (MS)
Base Station (BS / Cell Site)
Mobile Switching Center (MSC / MTSO)

Mobile Station (MS): User's device (e.g., phone). Contains control unit, transceiver, antenna. Can be vehicle-mounted or portable.

Base Station (BS): Interfaces MTSO and mobile units. Has control unit, radio cabinets, antennas, power plant, data terminals. Transmit power determines cell size. Bridges communication for users in its cell.

Mobile Switching Center (MSC/MTSO): Central coordinator for all cell sites. Includes cellular processor and switch. Interfaces with telephone company. Manages call processing, billing, handoffs. Connects mobile users to each other and to PSTN. Handles ~100,000 users, 5,000 simultaneous calls.

Links: Radio Links (Air Interface) between MS and BS for voice/signaling. High-Speed Data Links (microwave/T-carriers) between BS and MTSO for voice/data.

D. Common Air Interface (CAI) & Call Establishment

CAI: Protocol for MS-BS radio interaction. Defines signaling for control channels and speech/channel coding for reliability.

Channel Types:

  • FCC (Forward Control Channel): BS to MS setup.
  • RCC (Reverse Control Channel): MS to BS setup.
  • FVC (Forward Voice Channel): BS to MS voice.
  • RVC (Reverse Voice Channel): MS to BS voice.

Call Flow Visualization

B. Second Generation (2G) Cellular Networks

2G introduced digital technology, improving spectral efficiency and enabling services like SMS.

GSM (Global System for Mobile Communications):

  • Launched 1991. Bands: 935-960 MHz (Forward), 890-915 MHz (Reverse). 124 channels (200 KHz).
  • TDMA: 8 users/channel. GMSK modulation. Data rate: 270.8 Kbps. Voice: 13 kbps.
  • Features: SMS, MMS, roaming, data/fax (9.6 Mbps), CLIP.
Mobile Station (MS)
Base Station Subsystem (BSS)
Network Switching Subsystem (NSS)
Operation Support Subsystem (OSS)

MS (Mobile Station): Physical equipment + SIM card. Provides air interface for voice, data, SMS.

BSS (Base Station Subsystem): Consists of BTS (Base Transceiver Station) and BSC (Base Station Controller). BTS handles radio link protocols. BSC manages radio resources, handovers, connects to MSC.

NSS (Network Switching Subsystem): Core is MSC. Handles call switching, authentication, registration, location updating, routing. Includes HLR, VLR, AUC, EIR.

OSS (Operation Support Subsystem): Operations and Maintenance Center (OMC). Manages administration, security, configuration, maintenance.

Other 2G Systems: D-AMPS (IS-136), IS-95 (CDMA).

C. Wireless in Local Loop (WLL)

WLL replaces traditional wired connections (copper) with wireless radio links to connect subscribers to the local telephone exchange. It's cost-effective for rural or hard-to-wire areas.

Architecture: PSTN, WANU (Wireless Access Network Unit at exchange), Switch Function, WASU (Wireless Access Subscriber Unit at premises).

Advantages: No physical wiring, cost-effective, quick deployment, secure, scalable. Supports voice, internet, data, fax.

Disadvantages: Susceptible to interference, potentially lower bandwidth, limited range.

D. Wireless Local Area Networks (WLAN)

WLANs provide wireless connectivity within homes, buildings, offices, extending wired LANs. Typically low power, unlicensed spectrum (e.g., Wi-Fi IEEE 802.11).

Modes: Infrastructure (via Access Point), Ad Hoc (device-to-device).

Advantages: Fast installation, flexibility, mobility, reduced cost of ownership.

Disadvantages: Slower bandwidth than wired, security concerns, capacity limits.

Security: WPA3, OWE, MPSK. QoS via WMM.

E. Bluetooth and Personal Area Networks (PANs)

Bluetooth creates ad-hoc PANs for temporary connections in a personal workspace (nominal range 10m). Operates in 2.4 GHz ISM band, uses Frequency Hopping TDD.

Architecture:

  • Piconet: 1 Master, up to 7 active Slaves. Max 8 active nodes.
  • Scatternet: Interconnected Piconets. A slave in one can be master/slave in another (bridge slave).

Data Rate: Up to 24 Mbps (v3.0+).

Advantages: Low cost, ease of use, ad-hoc, fast connection, voice/data.

Disadvantages: Lower security, slower than WLAN, small range, no native routing.

Unit 3: Introduction to Cellular Mobile Systems

This unit delves into the specifics of cellular systems, covering spectrum allocation, basic system operations, performance criteria, and a comparison between analog and digital cellular technologies, including an overview of GSM architecture.

A. Spectrum Allocation

Spectrum allocation refers to the designation of specific frequency bands by regulatory bodies (like the FCC in the U.S. or ITU globally) for different wireless services, including mobile communications. Efficient allocation is crucial to prevent interference and maximize the use of this finite resource. Cellular systems operate in licensed bands, meaning operators pay for exclusive rights to use certain frequencies within a geographic area.

B. Basic Cellular Systems & Operation

Covered in Unit 1 (Basic Architecture & Call Establishment). The core idea is to divide a large service area into smaller cells, each served by a low-power base station. This allows for frequency reuse, significantly increasing system capacity. The MSC coordinates handoffs as users move between cells.

C. Performance Criteria

Key metrics for evaluating cellular system performance:

  • Voice Quality: Subjectively rated (e.g., Mean Opinion Score - MOS). Aim for toll quality (MOS ≥ 4).
  • Service Quality:
    • Coverage: Percentage of area with good signal (e.g., 90% in flat terrain).
    • Grade of Service (GOS): Probability of a call being blocked or delayed (e.g., 2% blocking during busy hour).
    • Dropped Calls: Rate of calls terminated unintentionally. Must be minimized.
  • Special Features: Availability of services like call forwarding, voicemail, roaming.

Unit 4: Cellular System Design Fundamentals

This unit covers essential design principles for cellular systems, including frequency reuse, channel assignment strategies, handoff mechanisms, interference management, and techniques for improving coverage and capacity.

A. Frequency Reuse

Frequency reuse is a core concept where the same set of frequencies is used in different geographic cells that are sufficiently separated to avoid unacceptable interference. This dramatically increases system capacity.

A cluster is a group of N cells where each cell uses a unique set of frequencies, and the entire set of system frequencies is used within the cluster. The cluster can then be repeated. Common cluster sizes (N) are 4, 7, 12. The frequency reuse factor is 1/N.

For hexagonal cells, $N = i^2 + ij + j^2$ (i, j are non-negative integers).

Simplified visualization of hexagonal cells and frequency groups.

B. Channel Assignment Strategies

  • Fixed Channel Assignment (FCA): Each cell is permanently allocated a predetermined set of voice channels. If all channels in a cell are occupied, a new call is blocked. Simple, but can be inefficient if traffic is uneven.
    • Borrowing Strategy: A congested cell can borrow channels from an adjacent cell.
  • Dynamic Channel Assignment (DCA): Voice channels are not permanently allocated to cells. When a call request is made, the serving base station requests a channel from the MSC. The MSC allocates a channel based on factors like frequency reuse constraints, signal strength, and traffic distribution. More complex, but can reduce blocking probability and increase trunking efficiency.

C. Handoff Strategies

Handoff (or handover) is the process of transferring an active call or data session from one cell to another or from one channel to another within the same cell as a mobile user moves. Essential for seamless communication.

  • Network-Controlled Handoff (NCHO): The network makes handoff decisions.
  • Mobile-Assisted Handoff (MAHO): The mobile station measures signal strength from surrounding base stations and reports this to the network, assisting in the handoff decision. Common in 2G systems like GSM.
  • Mobile-Controlled Handoff (MCHO): The mobile station makes the handoff decision.
  • Hard Handoff: "Break-before-make." The connection to the old base station is broken before the connection to the new one is established. Characteristic of FDMA and TDMA systems.
  • Soft Handoff: "Make-before-break." The mobile station temporarily connects to multiple base stations simultaneously. Characteristic of CDMA systems. Results in smoother transitions.

D. Interference and System Capacity

Interference is a major limiting factor. Capacity is related to how many users can be served with acceptable quality.

  • Co-channel Interference (CCI): Interference between cells using the same set of frequencies. Reduced by increasing distance between co-channel cells (larger N). S/I ratio is key.
  • Adjacent Channel Interference (ACI): Interference from signals in adjacent frequency channels due to imperfect filters. Minimized by careful channel assignment and filtering. Near-far effect is a severe form.

Power Control: Crucial for reducing interference (especially reverse channel CCI) and conserving battery life. Especially important in CDMA.

E. Tracking and Grade of Service (GOS)

Tracking: The process by which the system keeps track of a mobile user's location to route incoming calls and manage handoffs. Involves registration, location updates.

Grade of Service (GOS): A measure of the probability that a user will be denied access to the network (call blocked) or delayed. Typically defined as the blocking probability during the busy hour (e.g., 2%). Erlang B formula for blocked-calls-cleared systems, Erlang C for queued systems.

F. Improving Coverage and Capacity

  • Cell Splitting: Dividing congested cells into smaller cells (microcells), each with its own BS, lower power, and reduced antenna height. Increases frequency reuse.
  • Sectoring: Using directional antennas at the BS to divide a cell into sectors (e.g., 3x120° or 6x60°). Reduces CCI by limiting interference to fewer co-channel cells, allowing smaller N. Increases handoffs and can reduce trunking efficiency.
  • Repeaters: Amplify and reradiate signals to extend coverage into hard-to-reach areas (tunnels, valleys). Do not add capacity.
  • Microcell Zone Concept (Intelligent Cell): Multiple zone sites (Tx/Rx) connected to a single BS, sharing radio equipment. Mobile is served by the zone with the strongest signal on the same channel. Reduces MSC handoff load and CCI.
  • Umbrella Cells: Co-locating large and small cells. Large cells provide wide coverage for high-speed users (fewer handoffs), small cells provide capacity for low-speed users.

Unit 5: Multiple Access Techniques for Wireless Communication

This unit explains how multiple users share the limited radio spectrum. Key techniques include FDMA, TDMA, Spread Spectrum (CDMA), and SDMA.

A. Introduction to Multiple Access

Multiple access schemes allow many users to share a common transmission medium (radio spectrum). The main goal is to maximize the number of users while maintaining acceptable communication quality.

B. Multiple Access Techniques Overview

FDMA (Frequency Division Multiple Access)

Spectrum is divided into distinct frequency channels. Each user gets an exclusive frequency band for the duration of the call. Guard bands separate channels. Simple, but inefficient if channels are idle.

TDMA (Time Division Multiple Access)

Users share the same frequency band but are allocated different time slots. Data is transmitted in bursts. More spectrally efficient than FDMA. Requires synchronization.

CDMA (Code Division Multiple Access)

All users transmit simultaneously over the entire frequency band. Each user is assigned a unique pseudo-random code (spreading code) to differentiate their signal. Offers high capacity and soft handoff.

SDMA (Space Division Multiple Access)

Users are separated spatially using directional antennas or beamforming. Allows reuse of the same frequency/time/code in different spatial locations. Increases capacity and reduces interference.

C. Spread Spectrum Multiple Access (CDMA in detail)

Principles: Message signal is multiplied by a wideband pseudo-random spreading code. This spreads the signal's bandwidth. Processing gain = (Spread BW) / (Original BW). At receiver, signal is multiplied by same code to despread desired signal and spread interference.

Orthogonal Codes: Codes where the product of two distinct codes is zero, and product with itself is non-zero. Allows multiple users on same frequency without interference if power controlled.

Comparison with TDMA: CDMA generally offers higher capacity (N=1 frequency reuse), better interference management (with power control), superior voice quality (soft handoff), but is more complex.

B. Development of Wireless Networks & Fixed Network Hierarchy

Wireless networks evolved from basic coverage to complex systems integrated with the PSTN. The PSTN uses a digital signaling (DS) hierarchy (e.g., DS-0 for 64kbps voice, T-1/E-1 for multiple channels) for high-data-rate transport.

C. Traffic Routing in Wireless Networks

  • Connection-Oriented (Virtual Circuit): Fixed path established for duration of call (e.g., circuit switching for voice). Ordered data arrival. Inefficient for bursty data.
  • Connectionless (Datagram): No fixed path; data broken into packets, routed independently (e.g., packet switching for data). Packets may arrive out of order. Efficient for bursty data. X.25 is an early standard.

D. Wireless Data Services

Early services for data over mobile networks:

  • CDPD (Cellular Digital Packet Data): Used idle AMPS channels (19.2 kbps).
  • ARDIS: Private network (Motorola/IBM) for short messages (4.8-19.2 kbps).
  • RAM Mobile Data (RMD): Public service (Mobitex protocol) for data/fax (8 kbps).

E. Common Channel Signaling (CCS)

Transmits control/signaling data on a separate channel from user data (out-of-band signaling). SS7 (Signaling System No. 7) is a key CCS protocol.

Advantages: Faster communication, increased capacity (voice trunks for voice only), efficiency, simultaneous handling, reduced delay/interference.

SS7 Architecture: SSPs (Switches), STPs (Packet Switches/Routers), SCPs (Databases for call processing).

F. ISDN (Integrated Services Digital Network)

Network framework based on CCS, providing digital end-to-end connectivity for voice, data, video. Uses SS7 for network signaling.

  • Channels: B-channels (Bearer, user traffic), D-channels (Data/Signaling), H-channels (High-speed).
  • Interfaces: BRI (2B+D for small users), PRI (23B+D or 30B+D for large users).
  • B-ISDN & ATM: Broadband ISDN uses Asynchronous Transfer Mode for higher bandwidths (Mbps to Gbps), handling voice/data in fixed-length cells (53 bytes).

G. Intelligent Cell Concept

Discussed in Unit 4 (Improving Coverage & Capacity). Aims to reduce MSC handoff load and co-channel interference by using multiple low-power zone transmitters within a cell, all controlled by a single base station. The mobile communicates on the same frequency channel as it moves between zones, with the base station switching the active zone site.
